Known limitations at this checkpoint
- `com.hpr.fpsdemo` still contains most runtime execution logic and project-specific editor/bootstrap code
- `SceneBootstrap` still knows the current project scene/hierarchy
- only `com.hpr.stats`, `com.hpr.eventbus`, and `com.hpr.inventory` currently have committed standalone demo scenes
- stats/interaction/ui/world/bootstrap packages are not real productized packages yet
