# Games Repo Documentation

This folder is the handoff point for the Unity FPS project and its tooling.

Read in this order:
1. `handoff.md` - current status, risks, and exact takeover steps
2. `workflows.md` - build/import/smoke/asset workflows
3. `current-state.md` - architecture and repository layout
4. `local-assets.md` - real local Asset Store inventory and current integration state
5. `report.txt` - compact session report

Key rule for this repo:
- imported Unity/Asset Store data stays local-only and must not be committed
- gameplay code, tooling, docs, and reproducible setup scripts must be committed
